Documentación del Proyecto

Detalles completos sobre el sistema de gestión de la Selección Colombia.

1. Introducción

Este proyecto es un sistema de gestión para la Selección Colombia, diseñado para administrar jugadores, equipos rivales, partidos, torneos y plantillas de manera eficiente. La aplicación permite realizar operaciones CRUD (Crear, Leer, Actualizar, Eliminar) sobre cada modelo, además de ofrecer estadísticas detalladas y un historial de acciones.

2. Objetivos del Proyecto

2.1 Objetivo General

Desarrollar un sistema web que permita la gestión integral de la Selección Colombia, incluyendo el manejo de datos de jugadores, equipos, partidos, torneos y plantillas, con un enfoque en la usabilidad y la eficiencia.

2.2 Objetivos Específicos

  • Implementar operaciones CRUD para cada modelo (Jugadores, Equipos, Partidos, Torneos, Plantillas).
  • Proveer estadísticas detalladas sobre el rendimiento de la Selección Colombia en partidos.
  • Mantener un historial de acciones para cada modelo, permitiendo auditorías y recuperación de datos.
  • Diseñar una interfaz moderna y responsiva con estilos CSS personalizados.
  • Incorporar documentación clara y accesible dentro de la aplicación.

3. Descripción de los Modelos

Representa a un jugador de fútbol de la Selección Colombia.

  • id: Identificador único (entero).
  • Jugadores: Nombre del jugador (texto).
  • F_Nacim_Edad: Fecha de nacimiento y edad (texto, formato: "YYYY-MM-DD (edad)")
  • Club: Club actual del jugador (texto).
  • Altura: Altura del jugador (texto, formato: "X.XX m").
  • Pie: Pie dominante (texto: "Derecho", "Izquierdo", "Ambos").
  • Partidos_con_la_seleccion: Número de partidos jugados (entero).
  • Goles: Número de goles anotados (entero).
  • Numero_de_camisa: Número de camiseta (entero).
  • anio: Año de registro (entero, entre 2021 y 2025).
  • posicion: Posición en el campo (texto: "Portero", "Defensa", "Medio", "Delantero").
  • activo: Estado del jugador (booleano: True si está activo, False si no).

Representa a un equipo rival de la Selección Colombia.

  • id: Identificador único (texto).
  • nombre: Nombre del equipo (texto).
  • pais: País del equipo (texto).
  • enfrentamientos_con_colombia: Número de enfrentamientos (entero).

Representa un partido jugado por la Selección Colombia.

  • id: Identificador único (texto).
  • equipo_local: Nombre del equipo local (texto).
  • equipo_visitante: Nombre del equipo visitante (texto).
  • fecha: Fecha del partido (fecha, formato: "YYYY-MM-DD").
  • goles_local: Goles del equipo local (entero).
  • goles_visitante: Goles del equipo visitante (entero).
  • torneo_id: Identificador del torneo (texto).
  • eliminado: Equipo eliminado (texto, opcional).
  • tarjetas_amarillas_local: Tarjetas amarillas local (entero).
  • tarjetas_amarillas_visitante: Tarjetas amarillas visitante (entero).
  • tarjetas_rojas_local: Tarjetas rojas local (entero).
  • tarjetas_rojas_visitante: Tarjetas rojas visitante (entero).

Representa un torneo en el que participa la Selección Colombia.

  • id: Identificador único (texto).
  • nombre: Nombre del torneo (texto).
  • anio: Año del torneo (entero).
  • pais_anfitrion: País anfitrión (texto, opcional).
  • estado: Estado del torneo (texto: "En curso", "Finalizado", "Cancelado").
  • eliminado: Equipo eliminado (texto, opcional).

Representa la plantilla de un equipo para un torneo o año específico.

  • id: Identificador único (texto).
  • equipo_id: Identificador del equipo (texto).
  • nombre: Nombre del jugador (texto, opcional).
  • posicion: Posición del jugador (texto, opcional).
  • anio: Año de la plantilla (entero).

5. Estructura de la Aplicación

Tecnologías utilizadas:

  • Backend: FastAPI (Python) para rutas y lógica del servidor.
  • Frontend: Jinja2 para plantillas HTML, CSS profesional personalizado.
  • Almacenamiento: Archivos CSV con historial y papelera.
  • Estilo: Diseño responsivo con CSS moderno.

6. Instrucciones de Uso

  1. Instala dependencias: pip install fastapi uvicorn pandas pydantic.
  2. Ejecuta el servidor: uvicorn main:app --reload.
  3. Accede a http://127.0.0.1:8000.
  4. Navega por las secciones desde la barra de navegación.
  5. Usa los botones de "Crear", "Editar" y "Eliminar" para gestionar datos.
